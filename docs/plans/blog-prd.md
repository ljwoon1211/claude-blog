# 기술 블로그 PRD (Product Requirements Document)

## 개요

- **목적**: 개인 기술 블로그 — 어드민(본인)이 게시글을 작성·관리하고, 방문자는 열람 및 댓글 참여 가능
- **대상 사용자**: 어드민(블로그 주인), 방문자(일반 독자)
- **사이트 성격**: 기술 공유 + 포트폴리오 + 개인 브랜딩

---

## 1. 사이트 구조 (Navigation)

| 메뉴           | 설명                                               | 우선순위 |
| -------------- | -------------------------------------------------- | -------- |
| **홈**         | 랜딩 페이지 — 최신 글 하이라이트, 간단한 소개      | P0       |
| **자기소개**   | 프로필, 경력, 기술 스택, 연락처 (tiptap 편집 가능) | P0       |
| **포트폴리오** | 프로젝트 목록 — 태그 필터 + 검색                   | P0       |
| **공부**       | 학습 기록, 기술 노트 — 태그 필터 + 검색            | P0       |
| **회고**       | 월간/분기 회고, 행사 후기                          | P1       |

## 2. 권한 모델

| 역할       | 게시글 CRUD | 자기소개 수정 | 댓글 작성 | 게시글 열람 |
| ---------- | ----------- | ------------- | --------- | ----------- |
| **어드민** | ✅ (전체)   | ✅            | ✅        | ✅          |
| **방문자** | ❌          | ❌            | ✅        | ✅          |

### 인증 방식

- **어드민**: Supabase Auth (이메일/비밀번호 또는 OAuth)
  - 로그인 페이지: `/login` — URL 직접 입력으로만 접근 (로그인 버튼/링크 노출 없음)
  - 로그인 후 이전 페이지로 리다이렉트
- **방문자 댓글**: giscus (GitHub 계정으로 댓글)

### 어드민 인라인 UI (별도 /admin 페이지 없음)

- 어드민 로그인 상태에서만 보이는 인라인 버튼:
  - 게시글 상세: **[수정]** / **[삭제]** 버튼
  - 목록 페이지: **[새 글 작성]** 버튼
  - 자기소개 페이지: **[수정]** 버튼 → tiptap 에디터 모드 진입
  - 포트폴리오 상세: **[수정]** 버튼

## 3. 게시글 (Post)

### 3.1 데이터 모델

| 필드        | 타입      | 설명                                                        |
| ----------- | --------- | ----------------------------------------------------------- |
| `id`        | UUID      | PK                                                          |
| `title`     | string    | 게시글 제목                                                 |
| `slug`      | string    | URL-friendly 식별자 (자동 생성, 한국어 지원)                |
| `content`   | JSON      | tiptap 에디터 출력 (JSON 형태로 DB 저장)                    |
| `category`  | enum      | 카테고리 (`portfolio` / `study` / `retrospective` / `page`) |
| `thumbnail` | string?   | 썸네일 이미지 URL                                           |
| `published` | boolean   | 공개 여부 (false = 임시저장)                                |
| `createdAt` | timestamp | 생성일                                                      |
| `updatedAt` | timestamp | 수정일                                                      |
| `authorId`  | UUID      | FK → 어드민 사용자                                          |

> **참고**: 태그는 `post_tags` 중간 테이블로 관리 (Post에 tags 컬럼 없음)
> **참고**: 자기소개 페이지도 `category: 'page'`, `slug: 'about'`으로 posts 테이블에 통합

### 3.2 슬러그 리다이렉트 테이블

| 필드        | 타입      | 설명                       |
| ----------- | --------- | -------------------------- |
| `id`        | UUID      | PK                         |
| `oldSlug`   | string    | 이전 slug (unique, 인덱스) |
| `postId`    | UUID      | FK → posts                 |
| `createdAt` | timestamp | 리다이렉트 생성 시점       |

- slug 변경 시 이전 slug를 `slug_redirects`에 INSERT
- 이전 slug 접근 시 **301 리다이렉트** → 현재 slug로 이동
- 인덱스 기반 O(1) 조회로 성능 보장

### 3.3 에디터 — tiptap

- **라이브러리**: `@tiptap/react` + 확장 패키지
- **저장 형식**: tiptap JSON → DB(Supabase)에 저장
- **렌더링**: JSON → tiptap 뷰어 또는 HTML 변환

#### tiptap 확장 구성

| 확장                | 용도                                         |
| ------------------- | -------------------------------------------- |
| `StarterKit`        | 기본 텍스트 (heading, bold, italic, list 등) |
| `Image`             | 이미지 업로드 (복수, 드래그 앤 드롭)         |
| `Youtube`           | YouTube URL 붙여넣기 → 임베드                |
| `CodeBlockLowlight` | 코드 블록 + 신택스 하이라이팅                |
| `Placeholder`       | 빈 에디터 힌트 텍스트                        |
| `Link`              | 하이퍼링크                                   |

### 3.4 임시저장 & 자동저장

- `published: false` = 임시저장 상태
- **자동 저장**: 30초마다 변경사항이 있을 경우 자동 저장
  - 변경 감지: tiptap `onUpdate` 이벤트로 dirty flag 관리
  - 30초 interval → dirty flag가 true일 때만 저장 API 호출
  - 저장 성공 시 "자동 저장됨" 토스트 표시
- **임시저장 관리 UI**: 게시글 작성 페이지에 [임시저장 불러오기] 버튼
  - 클릭 시 모달 → 임시저장 목록 표시 (제목, 수정일)
  - 각 항목에 **[불러오기]** / **[삭제]** 버튼
  - 불러오기 시 현재 작성 중인 내용이 있으면 "현재 내용을 덮어쓰시겠습니까?" 확인 모달

### 3.5 슬러그(Slug) 관리

- 제목 → 자동 slug 생성 (**한국어 지원**: `slugify` + 한글 처리)
- slug 수동 수정 가능
- slug 변경 시 → `slug_redirects` 테이블에 이전 slug 기록

### 3.6 콘텐츠 기능

#### 이미지 업로드

- 복수 이미지 업로드 지원
- 저장소: **Cloudflare R2**
- 에디터 내에서 드래그 앤 드롭 / 파일 선택 / 클립보드 붙여넣기
- **제한**: 장당 10MB, 허용 포맷 jpg/png/webp/gif
- 초과 시 에러 토스트 표시

#### YouTube 임베드

- YouTube URL 붙여넣기 → tiptap `Youtube` 확장으로 자동 임베드
- 프론트 렌더링 시 `react-lite-youtube-embed`로 lazy loading 최적화

#### 코드 블록

- `lowlight` 기반 신택스 하이라이팅 (tiptap `CodeBlockLowlight`)
- 언어 선택 + 복사 버튼
- 포트폴리오에서 링크 표기 등 다양한 용도로 활용

### 3.7 확인 모달

| 동작                  | 모달 메시지                                                            |
| --------------------- | ---------------------------------------------------------------------- |
| **게시 (공개)**       | "게시글을 공개하시겠습니까?"                                           |
| **게시글 삭제**       | "정말 삭제하시겠습니까? 이 작업은 되돌릴 수 없습니다."                 |
| **임시저장 불러오기** | "현재 작성 중인 내용을 덮어쓰시겠습니까?" (작성 중인 내용이 있을 때만) |
| **임시저장 삭제**     | "이 임시저장을 삭제하시겠습니까?"                                      |

## 4. 이미지 라이프사이클 관리

> [!IMPORTANT]
> 이미지는 모든 시나리오에서 안전하게 정리되어야 합니다.

### 4.1 이미지 추적

| 필드        | 타입      | 설명                                    |
| ----------- | --------- | --------------------------------------- |
| `id`        | UUID      | PK                                      |
| `url`       | string    | R2 URL                                  |
| `key`       | string    | R2 오브젝트 키                          |
| `postId`    | UUID?     | FK → 연결된 게시글 (null = 고아 이미지) |
| `createdAt` | timestamp | 업로드 시점                             |

### 4.2 삭제 시나리오

| 시나리오                    | 동작                                                              |
| --------------------------- | ----------------------------------------------------------------- |
| **게시글 삭제**             | 해당 게시글의 모든 이미지를 R2에서 삭제 + DB 레코드 삭제          |
| **편집 중 이미지 제거**     | 저장 시 현재 content의 이미지 URL과 DB 비교 → 빠진 이미지 R2 삭제 |
| **업로드 실패 / 서버 오류** | 업로드 후 게시글 저장 실패 시 → 업로드된 이미지 롤백 삭제         |
| **고아 이미지 정리**        | `postId`가 null이고 생성 후 24시간 경과 → Inngest cron으로 정리   |

### 4.3 구현 방식

- **편집 중 이미지 제거**: 게시글 저장(update) 시 tiptap JSON에서 이미지 URL 추출 → DB 비교 → 삭제 대상 R2 정리
- **서버 오류 롤백**: 이미지 업로드 시 `postId: null`로 저장 → 게시글 저장 성공 시 `postId` 연결 → 미연결 이미지는 고아 정리 대상
- **고아 정리**: Inngest cron job (매일 1회) — 24시간 이상 `postId: null`인 이미지 일괄 삭제

## 5. 태그 시스템

### 5.1 동작 방식

- 어드민이 게시글 작성 시 **자유롭게 입력** → 새 태그 자동 생성
- 별도 태그 관리 페이지 불필요
- 예시: `next.js`, `프론트엔드`, `인프라`, `AWS`, `TypeScript`, `Docker`, `CI/CD` 등

### 5.2 데이터 모델

**tags 테이블**

| 필드   | 타입   | 설명            |
| ------ | ------ | --------------- |
| `id`   | UUID   | PK              |
| `name` | string | 태그명 (unique) |
| `slug` | string | URL용 (unique)  |

- `postCount`는 컬럼으로 관리하지 않음 → 쿼리 시 `COUNT(*)` 집계 (개인 블로그 규모에서 충분)

**post_tags 중간 테이블**

| 필드     | 타입 | 설명                 |
| -------- | ---- | -------------------- |
| `postId` | UUID | FK → posts (복합 PK) |
| `tagId`  | UUID | FK → tags (복합 PK)  |

### 5.3 UI

- 게시글 작성 시: **태그 입력 인풋** (콤마/엔터로 구분, 기존 태그 자동완성)
- 목록 페이지: 상단에 태그 칩 목록 → 클릭 시 해당 태그만 필터링

## 6. 목록 페이지 (포트폴리오 / 공부 / 회고)

### 6.1 레이아웃

```
┌─────────────────────────────────────────┐
│  [태그1] [태그2] [태그3] [태그4] ...    │  ← 태그 필터
├─────────────────────────────────────────┤
│  🔍  검색어를 입력하세요...              │  ← 검색 인풋
├─────────────────────────────────────────┤
│  ┌────────┐  ┌────────┐  ┌────────┐    │
│  │ Post 1 │  │ Post 2 │  │ Post 3 │    │  ← 게시글 카드 그리드
│  └────────┘  └────────┘  └────────┘    │
│  ┌────────┐  ┌────────┐  ...           │
│  │ Post 4 │  │ Post 5 │               │  ← 무한 스크롤
│  └────────┘  └────────┘               │
└─────────────────────────────────────────┘
```

### 6.2 기능

| 기능            | 설명                                     | 구현                                                      |
| --------------- | ---------------------------------------- | --------------------------------------------------------- |
| **태그 필터**   | 상단 태그 클릭 → 해당 태그 게시글만 표시 | URL 쿼리 파라미터 (`?tag=next.js`)                        |
| **검색**        | 제목 + 내용 + 태그 기반 검색             | Drizzle `ilike` 또는 Supabase Full-Text Search            |
| **무한 스크롤** | 스크롤 시 자동 로딩                      | TanStack Query `useInfiniteQuery` + Intersection Observer |
| **정렬**        | 최신순 (기본)                            | 쿼리 파라미터 (`?sort=latest`)                            |

### 6.3 페이지네이션 (Cursor 기반)

- Cursor 기반 → 무한 스크롤에 적합 (offset 대비 성능 우위)
- `limit`: 한 번에 가져올 개수 (기본 12)

## 7. 게시글 상세 페이지

### 7.1 목차 (Table of Contents)

- tiptap JSON에서 heading 노드(`h2`, `h3`) 추출 → 사이드바 TOC 생성
- 스크롤 위치에 따라 현재 섹션 하이라이트 (Intersection Observer)
- **데스크톱**: 우측 사이드바 고정 / **모바일**: 상단 토글 펼침

### 7.2 공유 기능

- 게시글 상단 또는 하단에 공유 버튼:
  - **링크 복사**: 클립보드 복사 + "복사됨" 토스트
  - **SNS**: X(Twitter), LinkedIn 공유 (URL 파라미터 방식, SDK 불필요)

### 7.3 댓글 — giscus

- GitHub Discussions 기반, 마크다운 지원, 리액션 지원
- 다크모드 자동 연동
- 게시글 하단에 배치

## 9. UX 핵심 요소

### 9.1 로딩 상태

| 위치                      | 전략                  | 구현                                            |
| ------------------------- | --------------------- | ----------------------------------------------- |
| **목록 페이지 초기 로딩** | 스켈레톤 UI           | PostCard 형태의 회색 플레이스홀더               |
| **무한 스크롤 추가 로딩** | 하단 스피너           | Intersection Observer 트리거 시 로딩 인디케이터 |
| **게시글 상세 로딩**      | 스켈레톤 UI           | 제목/본문 영역 플레이스홀더                     |
| **에디터 자동저장**       | 인라인 상태 표시      | "저장 중..." → "자동 저장됨 (HH:MM)"            |
| **이미지 업로드**         | 프로그레스 바         | 업로드 진행률 표시 + 블러 placeholder           |
| **페이지 전환**           | Next.js `loading.tsx` | 라우트별 스켈레톤                               |

### 9.2 에러 핸들링

| 상황                   | 처리                                                |
| ---------------------- | --------------------------------------------------- |
| **404 — 페이지 없음**  | 커스텀 404 페이지 (홈 이동 링크, 검색 제안)         |
| **500 — 서버 오류**    | 커스텀 500 페이지 (재시도 버튼, Sentry 자동 리포트) |
| **API 오류**           | 토스트 알림 (에러 메시지 표시 + 재시도 버튼)        |
| **네트워크 오류**      | 오프라인 배너 + 자동 재연결 시 데이터 refetch       |
| **인증 만료**          | 자동 감지 → 로그인 페이지 리다이렉트                |
| **이미지 업로드 실패** | 에디터 내 에러 표시 + 재업로드 버튼                 |
| **자동 저장 실패**     | "저장 실패" 경고 + 수동 저장 유도                   |

- 모든 에러는 Sentry로 자동 수집
- `error.tsx` (라우트별 Error Boundary) + 전역 에러 바운더리

## 10. 캐싱 전략

| 대상            | 전략                                 | 설명                               |
| --------------- | ------------------------------------ | ---------------------------------- |
| **게시글 목록** | `use cache` + on-demand revalidation | 글 작성/수정/삭제 시 revalidateTag |
| **게시글 상세** | `use cache` + on-demand revalidation | 해당 slug revalidate               |
| **태그 목록**   | Upstash Redis (10분 TTL)             | 태그 목록은 자주 변하지 않음       |
| **자기소개**    | `use cache` + on-demand revalidation | 수정 빈도 낮음                     |
| **이미지**      | Cloudflare R2 CDN 캐시               | 엣지 캐싱                          |

- 어드민이 글 작성/수정/삭제 시 → `revalidateTag()` 또는 `revalidatePath()` 호출

## 11. 비기능 요구사항

| 항목              | 요구사항                                           |
| ----------------- | -------------------------------------------------- |

| **반응형**        | 모바일 / 태블릿 / 데스크톱                         |
| **다크모드**      | 시스템 설정 연동 + 토글                            |
| **i18n**          | next-intl 기반 (한국어 기본, 영어 확장 가능)       |
| **이미지 최적화** | Next.js Image + Cloudflare R2 CDN                  |
| **접근성**        | WAI-ARIA, 키보드 네비게이션                        |

## 12. 기술 설계 요약

### DB 테이블
